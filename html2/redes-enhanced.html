🔌 Redes LAN Ethernet (802.3)

Características Fundamentales

  • Topología: Física en ESTRELLA, lógica en BUS
  • Método de acceso: CSMA/CD (Carrier Sense Multiple Access/Collision Detection)
  • Tramas: 64-1518 bytes (sin preámbulo)
  • Dirección MAC: 48 bits únicos por interfaz
  • Velocidades: 10 Mbps - 10 Gbps

  • Mínimo: 64 bytes (sin preámbulo)

  • Etiqueta 802.1Q: +4 bytes (máximo 1522)

🔗 Subniveles de Enlace 802.3

LLC (Logical Link Control)

  • Identifica protocolo de capa de red
  • Control de errores y flujo
  • Envía trama al subnivel MAC

MAC (Media Access Control)

  • Encapsulación: delimitación y direccionamiento
  • Detección de errores
  • Control de acceso al medio
  • Recuperación de colisiones

🌉 Bridges (Puentes)

Funcionamiento

Capturan tramas de un segmento y las reenvían selectivamente a otro segmento, operando a nivel MAC.

🎯 Objetivos:

  • Interoperabilidad: Redes heterogéneas (Ethernet ↔ Token Ring)
  • Medios físicos: Unión de diferentes medios (UTP ↔ FO)
  • Fiabilidad: Redundancia y tolerancia a fallos
  • Seguridad: Segmentación evita modo promiscuo

Bridge con Encaminamiento desde Origen

Las estaciones indican el camino que seguirá la trama

Bridge Transparente

No requiere modificación en las estaciones. Funcionamiento automático.

🔄 Switches (Conmutadores)

Ventajas sobre Bridges

Interconectan múltiples segmentos a nivel de enlace con mayor rendimiento y transmisión full-duplex.

🔀 Formas de Conmutación:

📦

Store-and-Forward

Almacena toda la trama antes de reenviar

⚡

Cut-Through

Reenvía inmediatamente al leer destino

🧩

Fragment-Free

Lee primeros 64 bytes antes de reenviar

🔀

Híbrido

Cambia según frecuencia de errores

🛣️ Routers (Encaminadores)

Características

  • Interconexión de redes LAN separadas físicamente
  • Separan dominios de colisión y dominios de broadcast
  • Utilizan esquema de direccionamiento IP
  • Opciones de filtrado y conmutación más potentes que switches

⚠️ Diferencias Clave

Dispositivo Nivel OSI Dominios
Bridge Enlace (MAC) Separa colisiones
Switch Enlace (MAC) Separa colisiones
Router Red (IP) Separa colisiones y broadcast

🏷️ VLAN (Redes Virtuales)

Definición

Agrupación lógica de dispositivos y estaciones definida en uno o más switches, permitiendo agrupación aunque estén separados físicamente.

🔧 Tipos de VLAN

VLAN de Nivel 1 (Puerto)

Port Switching: Se especifica qué puertos pertenecen a la VLAN. Es el método más común pero no permite movilidad.

VLAN de Nivel 2 (MAC)

Asignación por dirección MAC. Permite movilidad pero requiere asignación individual.

VLAN por Protocolo

Determinada por el campo tipo de la trama MAC (IPv4, IPv6, AppleTalk, IPX).

VLAN de Nivel 3 (Subred)

Por direcciones de subred. Subred virtual basada en IP.

VLAN por Aplicación

Una VLAN específica para cada aplicación de niveles superiores.

🏷️ Etiquetado 802.1Q

Características del Etiquetado

  • Tamaño: 4 bytes adicionales en la trama
  • Trama máxima: 1522 bytes (1518 + 4)
  • Ubicación: Entre MAC origen y tipo/longitud
  • MSTP: Multiple Spanning Tree Protocol para múltiples VLAN

🔧 VTP (VLAN Trunking Protocol)

Protocolo propietario de Cisco que permite gestión centralizada de todas las VLAN en la red.

🌳 STP (Spanning Tree Protocol)

IEEE 802.1d

Protocolo de nivel 2 que gestiona bucles por enlaces redundantes mediante la definición de un único camino entre dos nodos.

🔄 Estados de Puertos STP:

🚫

Bloqueo

Recibe BPDUs pero no las envía

👂

Escucha

Determina rutas al puente raíz. Procesa BPDUs

📚

Aprendizaje

Actualiza tabla MAC. Descarta tramas de datos

✅

Envío

Envía y recibe datos. Actualiza tabla MAC

❌

Desactivado

Puerto deshabilitado o con fallo

⚡ RSTP y MSTP

RSTP (IEEE 802.1w)

  • Rapid STP: Tiempo de convergencia reducido
  • Compatibilidad: Mantiene compatibilidad con STP
  • Puertos: 2048-4096 puertos soportados
  • Estados: Learning, Forwarding, Discarding

MSTP (IEEE 802.1s)

  • Multiple STP: Extensión para múltiples VLAN
  • Regiones: Grupos de switches con mismo ID
  • Instancias: Múltiples instancias STP por VLAN
  • Escalabilidad: Mejor para organizaciones grandes

📡 SNMP (Simple Network Management Protocol)

Características Generales

  • Capa: Protocolo de aplicación
  • Transporte: UDP
  • Puertos: 161 (gestión) y 162 (traps)
  • Modelo: Agentes ↔ Gestores

🔢 Versiones SNMP

SNMP v1

  • Modo: Pregunta-respuesta sobre UDP
  • Limitación: No permite recuperar grandes bloques
  • Operaciones:
    • GetRequest
    • GetNextRequest
    • Response
    • SetRequest
    • Trap

SNMP v2

  • Mejoras: Operaciones adicionales
  • Nuevas operaciones:
    • GetBulk (recuperación masiva)
    • Inform (confirmación de traps)
    • Report (informes de estado)

SNMP v3

  • Seguridad: Mecanismos avanzados
  • VACM: Views-Based Access Control Model
  • USM: User-based Security Model
  • Características: Autenticación y cifrado

📊 MIB y SMI

SMI (Structure of Management Information)

Lenguaje basado en ASN.1 para definir la estructura de la información de gestión.

MIB (Management Information Base)

  • MIB-I: Estándar básico
  • MIB-II: Específica de cada fabricante
  • MIB propietarias: Extensiones del fabricante

🔍 RMON (Remote MONitoring MIB)

Funcionalidad

MIB que permite al gestor delegar la monitorización en agentes especiales (sondas RMON), reduciendo el tráfico de red.

RMON v1

  • Monitorización nivel físico y enlace
  • Nivel 2 (MAC)
  • Funciona off-line

RMON v2

  • Monitorización nivel de red
  • Nivel 3 (IP) y aplicación
  • Chequeo de tráfico completo

SMON

  • Switched MONitoring
  • Gestión de dispositivos de red
  • Monitorización de VPNs

✅ Ventajas RMON

  • Funcionamiento off-line
  • Disminución del tráfico de red
  • Detección y reporte automático de fallos
  • Datos con valor agregado
  • Soporte para múltiples gestores

📚 Glosario de Términos Clave

CSMA/CD
Carrier Sense Multiple Access with Collision Detection. Método de acceso al medio utilizado por Ethernet.
Dominio de Colisión
Área de red donde pueden ocurrir colisiones entre tramas. Separado por bridges y switches.
Dominio de Broadcast
Área de red donde se propagan las tramas de broadcast. Separado por routers.
VLAN
Virtual Local Area Network. Agrupación lógica de dispositivos independiente de su ubicación física.
STP
Spanning Tree Protocol. Previene bucles en redes con enlaces redundantes.
SNMP
Simple Network Management Protocol. Protocolo para gestión y monitorización de redes.
MIB
Management Information Base. Base de datos de objetos gestionables en SNMP.
RMON
Remote MONitoring. Extensión de SNMP para monitorización remota mediante sondas.
Trunking
Método para transportar múltiples VLANs a través de un enlace usando etiquetado.
Jumbo Frames
Tramas Ethernet de tamaño superior al estándar (típicamente 9000 bytes MTU).

💡 Consejos para el Examen

🎯 Puntos Clave

  • 802.3: Estándar Ethernet
  • 48 bits: Tamaño dirección MAC
  • 64-1518 bytes: Rango trama Ethernet
  • CSMA/CD: Método acceso Ethernet
  • 161/162: Puertos SNMP
  • 4 bytes: Etiqueta 802.1Q
  • 9000 bytes: MTU Jumbo Frames

⚠️ Errores Comunes

  • Confundir topología física (estrella) con lógica (bus) en Ethernet
  • Mezclar puertos SNMP (161 gestión, 162 traps)
  • Olvidar que routers separan dominios broadcast, switches solo colisiones
  • Confundir STP (802.1d) con RSTP (802.1w) y MSTP (802.1s)
