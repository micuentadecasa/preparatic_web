Mantenimiento de Sistemas

📊 Resumen Ejecutivo: Mantenimiento de Sistemas

¿Qué es el Mantenimiento de Sistemas?

Capacidad del software para ser modificado, incluyendo correcciones, mejoras o adaptaciones del sistema después de su entrega para corregir defectos, mejorar rendimiento o adaptarse a cambios del entorno.

🎯 Objetivo Principal

Asegurar la continuidad operacional y evolución del software mediante modificaciones controladas y planificadas.

📈 Impacto Económico

Representa el mayor porcentaje de los costes totales del ciclo de vida del sistema (hasta 70-80%).

🔧 Tipos Principales

Correctivo, Adaptativo, Evolutivo y Perfectivo según la metodología MÉTRICA v.3.

⚙️ Técnicas Avanzadas

Ingeniería Inversa, Reestructuración, Ingeniería hacia delante y Reingeniería.

📋 Objetivos de Aprendizaje

  • Comprender el concepto y características de la mantenibilidad
  • Identificar y clasificar los diferentes tipos de mantenimiento
  • Analizar los factores que influyen en los costes de mantenimiento
  • Aplicar técnicas modernas de mantenimiento de software
  • Utilizar herramientas especializadas para el mantenimiento

🏗️ Concepto de Mantenibilidad

Definición Formal (ANSI-IEEE)

"Modificación de un producto software después de su entrega para corregir defectos, mejorar el rendimiento u otros atributos, o adaptar el producto a un entorno modificado."

🎯 Características Fundamentales de la Mantenibilidad

🔍 Analizabilidad

Capacidad del producto software para permitir el diagnóstico de deficiencias o causas de fallos, o la identificación de las partes que han de ser modificadas.

🔄 Cambiabilidad

Capacidad del producto software para permitir que una determinada modificación sea implementada.

⚖️ Estabilidad

Capacidad del producto software para evitar efectos inesperados debidos a las modificaciones del software.

✅ Comprobabilidad

Capacidad del producto software para permitir que el software modificado sea validado.

📏 Conformidad

Capacidad del producto software para cumplir estándares o convenciones relacionadas con la mantenibilidad.

💡 Importancia Práctica

La mantenibilidad es crucial porque determina:

  • El tiempo necesario para realizar modificaciones
  • El coste asociado a los cambios
  • El riesgo de introducir nuevos errores
  • La capacidad de evolución del sistema

🔧 Tipos de Mantenimiento según MÉTRICA v.3

🚨 Mantenimiento Correctivo

Definición: Cambios realizados para corregir errores del software.

Características:

  • Corrección de bugs y defectos
  • Solución de problemas funcionales
  • Urgencia variable según criticidad

Ejemplo: Corregir un error de cálculo en nóminas

🔄 Mantenimiento Adaptativo

Definición: Modificaciones que afectan a los entornos en los que el sistema opera.

Características:

  • Cambios en el entorno operativo
  • Nuevas versiones de SO o hardware
  • Adaptación a nuevas regulaciones

Ejemplo: Migración a nueva versión de base de datos

📈 Mantenimiento Evolutivo

Definición: Modificaciones para cubrir expansiones o cambios en las necesidades del usuario.

Características:

  • Nuevas funcionalidades
  • Ampliación de capacidades
  • Respuesta a nuevos requisitos

Ejemplo: Añadir módulo de facturación electrónica

⚡ Mantenimiento Perfectivo

Definición: Acciones para mejorar la calidad interna del sistema.

Características:

  • Optimización del rendimiento
  • Mejora de la estructura del código
  • Reducción de complejidad

Ejemplo: Refactorización para mejorar legibilidad

💰 Análisis de Costes de Mantenimiento

🎯 Dato Clave

Los costes de mantenimiento representan el mayor porcentaje de los costes totales del ciclo de vida de un sistema (típicamente 60-80%).

📊 Factores que Influyen en los Costes

🔧 Factores Relacionados con el Software

  • Calidad de la documentación: Mayor calidad = menor coste
  • Cohesión y acoplamiento de módulos: Alta cohesión, bajo acoplamiento = menor coste
  • Lenguaje de programación: Lenguajes de alto nivel facilitan mantenimiento
  • Configuración del producto: Arquitectura modular reduce costes
  • Calidad de las pruebas: Mejor testing = menor coste correctivo

🏢 Factores Externos

  • Rotación del personal: Mayor rotación = mayor coste de aprendizaje
  • Dependencia del entorno de desarrollo: Entornos complejos aumentan costes
  • Disponibilidad de herramientas: Mejores herramientas reducen tiempo
  • Conocimiento del dominio: Experiencia reduce tiempo de análisis

📐 Fórmula TCA de Boehm

EMA = TCA × E

Donde:

  • EMA: Esfuerzo de Mantenimiento Anual
  • TCA: Tráfico de Cambio Anual (% del sistema que se modifica)
  • E: Esfuerzo de Desarrollo inicial

Ejemplo: Si un sistema requirió 100 personas-mes para desarrollarse y anualmente se modifica el 15%, el mantenimiento requerirá 15 personas-mes anuales.

📈 Distribución Típica de Costes

  • Correctivo: 15-20%
  • Adaptativo: 20-25%
  • Evolutivo: 50-60%
  • Perfectivo: 5-10%

⏱️ Factores de Tiempo

  • Urgencia de los cambios
  • Complejidad del análisis
  • Tiempo de pruebas
  • Coordinación con usuarios

🛠️ Técnicas Avanzadas de Mantenimiento

🔍 Ingeniería Inversa (Reverse Engineering)

Definición: Proceso de análisis de un sistema para identificar sus componentes y sus interrelaciones, creando representaciones del sistema en otra forma o en un nivel de abstracción más alto.

Aplicaciones:
  • Recuperación de documentación perdida
  • Comprensión de sistemas legacy
  • Análisis de arquitectura

🏗️ Reestructuración (Restructuring)

Definición: Transformación de una representación a otra en el mismo nivel de abstracción, preservando el comportamiento externo del sistema.

Objetivos:
  • Mejorar la estructura del código
  • Facilitar futuras modificaciones
  • Optimizar el rendimiento

⚡ Ingeniería hacia Delante (Forward Engineering)

Definición: Proceso tradicional que va desde abstracciones de alto nivel y diseños lógicos hacia la implementación física del sistema.

Características:
  • Desarrollo dirigido por modelos
  • Generación automática de código
  • Transformación de especificaciones

🔄 Reingeniería (Reengineering)

Definición: Examen y modificación de un sistema para reconstruirlo en una nueva forma, incluyendo ingeniería inversa, reestructuración e ingeniería hacia delante.

Proceso Completo:
  • Análisis del sistema existente
  • Rediseño de arquitectura
  • Implementación mejorada

🧰 Herramientas de Mantenimiento

🔍 Herramientas de Ingeniería Inversa

  • Analizadores de código estático
  • Extractores de arquitectura
  • Visualizadores de dependencias
  • Generadores de documentación

🏗️ Herramientas de Reestructuración

  • Refactoring automático
  • Optimizadores de código
  • Transformadores sintácticos
  • Analizadores de complejidad

🔄 Herramientas de Reingeniería

  • Plataformas integradas de reingeniería
  • Convertidores de lenguajes
  • Migradores de datos
  • Sistemas de transformación

📊 Herramientas de Análisis

  • Analizadores de métricas
  • Detectores de patrones
  • Evaluadores de calidad
  • Generadores de reportes

📚 Glosario de Términos Clave

Mantenibilidad

Capacidad del software para ser modificado, incluyendo correcciones, mejoras o adaptaciones después de su entrega.

Analizabilidad

Capacidad del producto software para permitir el diagnóstico de deficiencias y la identificación de partes a modificar.

Cambiabilidad

Capacidad del producto software para permitir que una determinada modificación sea implementada.

Estabilidad

Capacidad del producto software para evitar efectos inesperados debidos a las modificaciones.

Comprobabilidad

Capacidad del producto software para permitir que el software modificado sea validado.

Mantenimiento Correctivo

Cambios realizados para corregir errores del software detectados durante su operación.

Mantenimiento Adaptativo

Modificaciones que afectan a los entornos en los que el sistema opera (hardware, software base, regulaciones).

Mantenimiento Evolutivo

Modificaciones para cubrir expansiones o cambios en las necesidades del usuario, añadiendo nuevas funcionalidades.

Mantenimiento Perfectivo

Acciones para mejorar la calidad interna del sistema sin cambiar su funcionalidad externa.

Ingeniería Inversa

Proceso de análisis de un sistema para identificar componentes e interrelaciones, creando representaciones en niveles de abstracción más altos.

Reestructuración

Transformación de una representación a otra en el mismo nivel de abstracción, preservando el comportamiento externo.

Reingeniería

Examen y modificación de un sistema para reconstruirlo en una nueva forma, combinando múltiples técnicas de mantenimiento.

TCA (Tráfico de Cambio Anual)

Porcentaje del sistema que se modifica anualmente, utilizado en la fórmula de Boehm para calcular esfuerzo de mantenimiento.

Cohesión

Grado en que los elementos de un módulo están relacionados funcionalmente. Alta cohesión facilita el mantenimiento.

Acoplamiento

Grado de interdependencia entre módulos. Bajo acoplamiento facilita el mantenimiento y reduce costes.

MÉTRICA v.3

Metodología española para el desarrollo de sistemas de información que define los procesos de mantenimiento.
