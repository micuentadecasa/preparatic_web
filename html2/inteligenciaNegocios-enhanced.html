🏗️ Data Warehouse

Características OVNI (Inmon):

  • Orientado a temas
  • Variable en el tiempo
  • No volátil
  • Integrado

Sistemas Relacionados:

  • DSS (Decision Support System)
  • EIS (Executive Information System)
  • MIS (Management Information Systems)

🌊 Data Lake

Componentes Principales:

  • Ingesta de datos (batch, streaming, real-time)
  • Almacenamiento escalable
  • Seguridad y autorización
  • Análisis y machine learning
  • Gobierno de datos

Estructura por Capas:

  • Raw/Bronze: Datos en bruto
  • Cleansed/Silver: Datos filtrados
  • Curated/Gold: Datos optimizados
  • Laboratory: Experimentación

🔄 Data Lakehouse

Combina la flexibilidad de un Data Lake con las capacidades de gestión de un Data Warehouse.

Características Clave:

  • Tipos de datos extendidos
  • Transmisión de datos en tiempo real
  • Aplicación de esquemas
  • Soporte de BI y Data Science
  • Compatibilidad con transacciones
  • Formatos abiertos (Parquet)
  • Desacoplamiento procesamiento/almacenamiento
  • Soporte para R y Python
  • Hardware económico

🕸️ Data Fabric

Arquitectura que centraliza, conecta, gestiona y gobierna datos entre diferentes sistemas.

Componentes:

  • Conectores de datos
  • Gestión de datos
  • Modelado y capa semántica
  • Procesamiento y analíticas
  • Automatización

🌐 Data Mesh

Enfoque sociotécnico descentralizado para gestión de datos analíticos.

4 Principios Fundamentales:

  • Propiedad orientada por dominio
  • Datos como producto
  • Infraestructura de autoservicio
  • Gobernanza federada

📊 Sistemas OLAP

Procesamiento analítico en línea para consultas complejas e iterativas.

Tipos de OLAP:

  • MOLAP: Multidimensional OLAP
  • ROLAP: Relational OLAP
  • HOLAP: Hybrid OLAP
  • DOLAP: Desktop OLAP
  • SOLAP: Spatial OLAP
  • RTOLAP: Real Time OLAP
  • WOLAP: Web OLAP
  • In-memory OLAP: En memoria

🎯 Operaciones en Cubos OLAP

Operaciones de Navegación:

  • DRILL-DOWN: Añadir nivel de detalle
  • DRILL-UP: Reducir nivel de detalle
  • DRILL-ACROSS: Nueva dimensión
  • ROLL-ACROSS: Quitar dimensión

Operaciones de Visualización:

  • PIVOT-ROTATE: Cambiar perspectiva
  • SLICE: Seleccionar elemento
  • DICE: Seleccionar múltiples dimensiones
  • PAGE: Dividir en secciones

🏗️ Esquemas Multidimensionales

⭐ Esquema en Estrella

Una tabla de hechos central conectada a tablas de dimensiones.

  • Estructura simple
  • Consultas rápidas
  • Fácil de entender

❄️ Esquema Copo de Nieve

Esquema estrella con dimensiones normalizadas.

  • Menos redundancia
  • Más complejo
  • Consultas más lentas

🌌 Esquema Constelación

Múltiples esquemas estrella interconectados.

  • Múltiples hechos
  • Dimensiones compartidas
  • Muy flexible

🔐 Protección de la Confidencialidad

La confidencialidad garantiza que la información esté accesible únicamente a personal autorizado.

Marco Normativo:

  • ISO/IEC 27002: Garantizar acceso solo a autorizados
  • RGPD (UE) 2016/679: Seguridad adecuada de datos personales
  • LOPDGDD: Deber de confidencialidad en el tratamiento

🛡️ Privacy Enhancing Technologies (PET)

Sistemas que engloban procesos y métodos para protección de la intimidad y datos personales.

🎭 Herramientas de Ofuscación

  • Pruebas de conocimiento cero (ZKP)
  • Privacidad diferencial
  • Datos sintéticos
  • Anonimización y seudonimización

🔒 Procesamiento Encriptado

  • Encriptación homomórfica
  • Computación de múltiples partes
  • Intersección de conjuntos privados
  • Entornos de ejecución confiables

📊 Análisis Federado

Permite ejecutar tareas analíticas sobre datos no visibles para quien ejecuta las tareas.

  • Análisis distribuido
  • Preservación de privacidad
  • Colaboración segura

📋 Rendición de Cuentas

Sistemas para el control y auditoría del uso de datos.

  • Sistemas de rendición de cuentas
  • Umbrales de secretos compartidos
  • Almacenamiento de datos personales

📚 Glosario de Términos Clave

Business Intelligence (BI)
Conjunto de metodologías, aplicaciones, prácticas y capacidades enfocadas a la creación y administración de información para la toma de decisiones.
Data Lake
Repositorio de almacenamiento centralizado que contiene gran cantidad de datos en bruto, utilizando una arquitectura plana.
Data Warehouse
Base de datos corporativa caracterizada por la integración y depuración de información procedente de múltiples fuentes de datos.
Data Lakehouse
Arquitectura híbrida que combina las ventajas de flexibilidad de un Data Lake con las características de gestión de un Data Warehouse.
Data Fabric
Arquitectura de servicios y funcionalidades que contribuye a procesar mejor los volúmenes de datos de múltiples fuentes bajo una misma administración.
Data Mesh
Enfoque sociotécnico descentralizado para compartir, acceder y gestionar datos analíticos en entornos complejos y a gran escala.
ETL
Extract, Transform, Load - Proceso de extracción, transformación y carga de datos en un sistema de almacenamiento.
OLAP
Online Analytical Processing - Sistema para agilizar consultas de grandes cantidades de datos mediante análisis multidimensional.
PET
Privacy Enhancing Technologies - Tecnologías que protegen la intimidad y privacidad de los datos de las personas físicas.
Schema on Read
Estructura de datos donde el esquema no está predeterminado antes del almacenamiento, sino que se define al leer los datos.
Data Silos
Situación donde no existe un lugar centralizado para almacenar todos los datos de la organización, dificultando el descubrimiento de datos.
IDSA
International Data Spaces Association - Coalición de 133 empresas internacionales que trabaja en el concepto de espacio de datos.
Staging Area
Área temporal donde se almacenan los datos de origen para las cargas antes de ser transferidos al Data Warehouse.
Data Mart
Almacén de datos especializado por áreas o temas, subconjunto de los datos guardados en un Data Warehouse.
Cubo OLAP
Estructura multidimensional que convierte datos planos en una estructura de N dimensiones para análisis eficiente.

❓ Preguntas Frecuentes

¿Cuál es la diferencia principal entre Data Lake y Data Warehouse?

El Data Warehouse almacena datos estructurados con esquemas predefinidos (Schema on Write), mientras que el Data Lake puede almacenar cualquier tipo de datos con esquemas flexibles (Schema on Read).

¿Cuándo debería usar cada tipo de arquitectura?

Data Warehouse: Para análisis de BI tradicional con datos estructurados.
Data Lake: Para big data, IoT, y análisis de datos no estructurados.
Data Lakehouse: Cuando necesitas lo mejor de ambos mundos.

¿Qué es más importante: Data Fabric o Data Mesh?

Depende de la organización. Data Fabric es más técnico y centralizado, ideal para integración rápida. Data Mesh es organizativo y descentralizado, mejor para empresas grandes con dominios independientes.

¿Son obligatorias las tecnologías PET?

Con el RGPD y LOPDGDD, implementar medidas de protección de datos es obligatorio. Las PET son herramientas recomendadas para cumplir estos requisitos legales.

¿Cómo se relacionan los espacios de datos con la soberanía digital?

Los espacios de datos promueven la soberanía digital permitiendo a las organizaciones mantener el control sobre sus datos mientras participan en ecosistemas colaborativos seguros.
