Redes IP: IPv4 e IPv6 - Guía de Estudio

📋 Resumen Ejecutivo

Este tema abarca la arquitectura de redes IP, desde los fundamentos de IPv4 hasta las características avanzadas de IPv6 y los mecanismos de transición entre ambos protocolos.

📊 Datos Clave

  • IPv4: 32 bits = 4.3 mil millones de direcciones
  • IPv6: 128 bits = 2^128 direcciones
  • Cabecera IPv4: 20-60 bytes (variable)
  • Cabecera IPv6: 40 bytes (fija)
  • CIDR: Notación /n para máscaras

🎯 Conceptos Principales

  • Arquitectura TCP/IP
  • Direccionamiento y subnetting
  • Protocolos de enrutamiento
  • Calidad de servicio (QoS)
  • Transición IPv4-IPv6

🎓 Objetivos de Aprendizaje

  1. Comprender la estructura y funcionamiento del protocolo IPv4
  2. Dominar técnicas de direccionamiento y subnetting
  3. Analizar protocolos de enrutamiento interno y externo
  4. Evaluar las ventajas y características de IPv6
  5. Implementar estrategias de transición IPv4-IPv6

🌐 Protocolo IPv4 - Fundamentos

IPv4 es un protocolo sin conexión y de mejor esfuerzo que opera en la capa 3 del modelo OSI.

📋 Cabecera IPv4

Version (4 bits) | IHL (4 bits) | ToS (8 bits) | Total Length (16 bits) Identification (16 bits) | Flags (3 bits) | Fragment Offset (13 bits) TTL (8 bits) | Protocol (8 bits) | Header Checksum (16 bits) Source Address (32 bits) Destination Address (32 bits) Options (variable) | Padding
  • Tamaño mínimo: 20 bytes
  • Tamaño máximo: 60 bytes
  • IHL: Internet Header Length
  • TTL: Time To Live

📍 Direccionamiento IPv4

  • 32 bits divididos en 4 octetos
  • Notación decimal punteada: 192.168.1.1
  • Estructura: NET_ID + HOST_ID
  • Máscara de subred: Define límites de red

Clases de Direcciones (Classfull)

Clase Rango Máscara Hosts
A 1.0.0.0 - 126.255.255.255 /8 16,777,214
B 128.0.0.0 - 191.255.255.255 /16 65,534
C 192.0.0.0 - 223.255.255.255 /24 254

🔢 CIDR y Subnetting

CIDR (Classless Inter-Domain Routing) permite un direccionamiento más flexible que las clases tradicionales.

Ejemplo: 192.168.1.0/24 - Red: 192.168.1.0 - Máscara: 255.255.255.0 (/24) - Hosts disponibles: 2^8 - 2 = 254 - Primera IP útil: 192.168.1.1 - Última IP útil: 192.168.1.254 - Broadcast: 192.168.1.255

Fórmula de cálculo: Número de hosts = 2^n - 2 (donde n = bits de host)

🔧 Protocolos de Control

📡 ICMP (Internet Control Message Protocol)

Protocolo de control y diagnóstico que opera sobre IP.

  • Echo Request/Reply: ping
  • Destination Unreachable: Destino inalcanzable
  • Time Exceeded: TTL expirado
  • Redirect: Redirección de ruta

🔍 ARP/RARP

ARP (Address Resolution Protocol): Resuelve direcciones IP a MAC

RARP (Reverse ARP): Resuelve direcciones MAC a IP

Proceso ARP: 1. Host A quiere comunicarse con IP B 2. Verifica si B está en caché ARP 3. Si no, envía ARP Request broadcast 4. Host B responde con ARP Reply (unicast) 5. Host A actualiza tabla ARP

🌐 DHCP (Dynamic Host Configuration Protocol)

Asignación automática de configuración de red.

Métodos de Asignación:

  • Manual: Reserva específica por MAC
  • Automática: Asignación permanente desde pool
  • Dinámica: Asignación temporal con lease time

🛣️ Protocolos de Enrutamiento

IGP (Internal Gateway Protocols)

Protocolo Tipo Métrica Características
RIP v1/v2 Distance Vector Hop Count Max 15 hops, convergencia lenta
IGRP/EIGRP Distance Vector Compuesta Cisco propietario, convergencia rápida
OSPF Link State Cost Estándar, escalable, áreas jerárquicas

EGP (External Gateway Protocols)

  • EGP: Protocolo original, obsoleto
  • BGP: Border Gateway Protocol, actual estándar de Internet

🔀 NAT (Network Address Translation)

Traducción de direcciones IP para conservar direcciones públicas.

Tipos de NAT:

  • Static NAT: Mapeo 1:1 fijo
  • Dynamic NAT: Mapeo 1:1 desde pool
  • PAT (Port Address Translation): Mapeo many:1 con puertos

🚀 IPv6 - Fundamentos

IPv6 fue diseñado para resolver las limitaciones de IPv4, principalmente el agotamiento de direcciones.

📊 Características Principales

  • 128 bits = 2^128 direcciones
  • Cabecera fija de 40 bytes
  • No broadcast (unicast, anycast, multicast)
  • Multicast obligatorio
  • Autoconfiguración SLAAC
  • QoS nativo
  • IPSec integrado

📍 Formato de Direcciones IPv6

Formato completo: 2001:0db8:85a3:0000:0000:8a2e:0370:7334 Reglas de abreviación: 1. Eliminar ceros iniciales: 2001:db8:85a3:0:0:8a2e:370:7334 2. Comprimir ceros consecutivos (::): 2001:db8:85a3::8a2e:370:7334 3. Solo un :: por dirección Ejemplos: - ::1 (loopback) - :: (all zeros) - 2001:db8::/32 (documentation prefix)

🔧 Cabecera IPv6

Version (4) | Traffic Class (8) | Flow Label (20) Payload Length (16) | Next Header (8) | Hop Limit (8) Source Address (128 bits) Destination Address (128 bits)

Diferencias con IPv4:

  • Tamaño fijo: Siempre 40 bytes
  • No fragmentación en routers: Solo en hosts origen
  • Extension Headers: Funcionalidad opcional
  • Hop Limit: Reemplaza TTL

🔄 Transición IPv4-IPv6

Mecanismos para la coexistencia y migración entre protocolos.

Estrategias Principales:

📱 Dual Stack

Implementación simultánea de IPv4 e IPv6 en el mismo dispositivo.

  • Configuración paralela
  • Selección automática de protocolo
  • Compatibilidad completa

🚇 Tunneling

Encapsulación de IPv6 sobre IPv4 (o viceversa).

  • 6to4: Automático, usa 2002::/16
  • Teredo: Atraviesa NAT
  • ISATAP: Intra-Site Automatic
  • 6in4: Túnel manual configurado

🔄 Traducción (NAT64)

Traducción de protocolos para comunicación entre redes IPv4 e IPv6.

  • Stateful NAT64: Con tabla de estado
  • Stateless NAT64: Mapeo directo
  • DNS64: Traduce consultas DNS

🎯 Calidad de Servicio (QoS)

Mecanismos para garantizar rendimiento de aplicaciones críticas.

📊 Parámetros QoS

  • Bandwidth: Ancho de banda garantizado
  • Latency: Retardo máximo
  • Jitter: Variación del retardo
  • Packet Loss: Pérdida de paquetes

🔧 Mecanismos QoS

  • Classification: Identificación de tráfico
  • Marking: Etiquetado de paquetes
  • Queuing: Gestión de colas
  • Shaping: Control de velocidad
  • Policing: Limitación de tráfico

📡 Multicast

Transmisión uno-a-muchos eficiente para aplicaciones como streaming.

IPv4 Multicast:

  • Rango: 224.0.0.0/4 (Class D)
  • Protocolos: IGMP, PIM, DVMRP
  • Direcciones especiales: 224.0.0.1 (all hosts), 224.0.0.2 (all routers)

IPv6 Multicast:

  • Prefijo: FF00::/8
  • Obligatorio: Todos los nodos deben soportarlo
  • Reemplaza broadcast: No existe broadcast en IPv6

🔐 Seguridad en Redes IP

🛡️ IPv4 Security

  • IPSec: Opcional, implementación externa
  • Firewalls: Control de acceso por paquetes
  • VPN: Túneles seguros

🔒 IPv6 Security

  • IPSec integrado: Soporte nativo obligatorio
  • Autoconfiguración segura: SEND protocol
  • Privacy Extensions: Direcciones temporales

📚 Glosario de Términos

ARP (Address Resolution Protocol)
Protocolo que resuelve direcciones IP a direcciones MAC en redes locales.
BGP (Border Gateway Protocol)
Protocolo de enrutamiento externo utilizado para intercambio de información entre sistemas autónomos en Internet.
CIDR (Classless Inter-Domain Routing)
Sistema de direccionamiento que permite subredes de tamaño variable mediante notación /n.
DHCP (Dynamic Host Configuration Protocol)
Protocolo para asignación automática de configuración IP a dispositivos de red.
Dual Stack
Implementación simultánea de IPv4 e IPv6 en el mismo dispositivo o red.
ICMP (Internet Control Message Protocol)
Protocolo de control utilizado para diagnóstico y reporte de errores en redes IP.
IGP (Interior Gateway Protocol)
Protocolos de enrutamiento utilizados dentro de un sistema autónomo (RIP, OSPF, EIGRP).
NAT (Network Address Translation)
Técnica para traducir direcciones IP privadas a públicas, conservando el espacio de direcciones.
OSPF (Open Shortest Path First)
Protocolo de enrutamiento de estado de enlace que utiliza el algoritmo de Dijkstra.
PAT (Port Address Translation)
Tipo de NAT que permite múltiples hosts internos compartir una dirección IP pública usando diferentes puertos.
QoS (Quality of Service)
Conjunto de tecnologías para garantizar rendimiento de red para aplicaciones críticas.
RIP (Routing Information Protocol)
Protocolo de enrutamiento de vector distancia que utiliza conteo de saltos como métrica.
SLAAC (StateLess Address AutoConfiguration)
Mecanismo de IPv6 que permite autoconfiguración automática de direcciones sin servidor DHCP.
Subnetting
Técnica de dividir una red IP en subredes más pequeñas para optimizar el direccionamiento.
TTL (Time To Live)
Campo de la cabecera IPv4 que limita la vida útil de un paquete en la red.
Tunneling
Técnica de encapsular un protocolo dentro de otro para transporte a través de redes incompatibles.
