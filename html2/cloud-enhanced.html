Cloud Computing: IaaS, PaaS, SaaS

📋 Resumen Ejecutivo

Cloud Computing es un modelo que permite el acceso bajo demanda y a través de la red a un conjunto de recursos compartidos y configurables que pueden ser rápidamente asignados y liberados con una mínima gestión por parte del proveedor del servicio.

🎯 Objetivos de Aprendizaje

  • Comprender los conceptos fundamentales del Cloud Computing
  • Diferenciar entre IaaS, PaaS y SaaS
  • Conocer los tipos de nubes: pública, privada, híbrida y comunitaria
  • Identificar los principales proveedores y sus servicios
  • Entender la normativa aplicable en España

🔧 Características Esenciales del Cloud Computing

Según el NIST, el cloud computing tiene cinco características fundamentales:

🔄 Autoservicio bajo demanda

El cliente puede solicitar recursos sin interacción personal con el proveedor

Ejemplo: Crear una instancia virtual en AWS EC2 desde la consola web

🌐 Amplio acceso a través de redes

Acceso estándar a través de la red para todo tipo de dispositivos (PCs, tablets, smartphones)

📊 Agrupación de recursos

Los recursos se encuentran agrupados para múltiples clientes, con asignación dinámica

⚡ Elasticidad rápida

La capacidad es escalable rápidamente de acuerdo con la demanda requerida

📋 Servicio medido

El uso de recursos puede ser medido, monitorizado y controlado de modo transparente

💡 Ventajas del Cloud Computing

Facilidad y rapidez: No requiere instalación, accesible desde cualquier lugar
Ahorro: En hardware y licencias con sistema de pago por uso
Oferta de servicios: Más sencillo encontrar servicios que desarrollarlos
Escalabilidad: Ajuste dinámico de recursos según necesidades

⚠️ Desventajas del Cloud Computing

Dependencia de Internet: Requiere conexión estable para acceso
Vendor Lock-In: Dependencia de proveedores específicos
Seguridad y privacidad: Desconfianza al almacenar datos externamente
Personalización limitada: Menos flexible que soluciones on-premise

☁️ Tipos de Infraestructura Cloud

La infraestructura se clasifica según su forma de despliegue en cuatro tipos principales:

🌐 Nube Pública

Infraestructura y recursos disponibles para el público general a través de Internet

Ejemplos: AWS, Azure, Google Cloud

✅ Ventajas

  • Escalabilidad y flexibilidad
  • Reducción de costos y pagos por uso
  • Actualizaciones automáticas

❌ Inconvenientes

  • Limitaciones en personalización
  • Dependencia del proveedor
  • Riesgo de interrupciones

🏢 Nube Privada

Infraestructura controlada y operada en beneficio exclusivo de una organización

Ejemplo: NubeSARA (promovida por la SGAD)

✅ Ventajas

  • Mayor seguridad y privacidad
  • Mayor control y personalización
  • Integración con sistemas heredados

❌ Inconvenientes

  • Mayor costo y personal especializado
  • Menor flexibilidad y escalabilidad
  • Mantenimiento manual

🤝 Nube Comunitaria

Alianza de organizaciones con objetivos similares y marco de seguridad común

Ejemplo destacado: GAIA-X (iniciativa europea)

🔀 Nube Híbrida

Utilización conjunta de varias infraestructuras cloud mantenidas como entidades separadas pero unidas por tecnología

Composición: Al menos una nube privada + una nube pública

Implementación típica:

  • Proveedor con nube privada forma alianza con proveedor público
  • Proveedor público forma alianza con proveedor de nube privada

🛠️ Tipos de Servicios Cloud

Los servicios cloud se clasifican en tres tipos principales según el nivel de abstracción:

🏗️ IaaS - Infrastructure as a Service

Entrega de recursos informáticos como servidores, almacenamiento y redes a través de internet

Responsabilidades:
• Proveedor: Seguridad física de la infraestructura
• Cliente: Sistema operativo, aplicaciones y datos

Ejemplos:

  • Propietarios: Amazon EC2, Google Compute Engine, Azure Virtual Machines
  • Open Source: OpenStack, Apache CloudStack, OpenNebula

🎯 PaaS - Platform as a Service

Plataforma completa de desarrollo y despliegue sin gestión de infraestructura

Responsabilidades:
• Proveedor: Plataforma y entorno de ejecución
• Cliente: Aplicaciones y datos desarrollados

Ejemplos:

  • Propietarios: Google App Engine, AWS Elastic Beanstalk, Azure App Service
  • Open Source: Heroku, Red Hat OpenShift, Cloud Foundry

💼 SaaS - Software as a Service

Aplicaciones de software completas entregadas a través de internet

Responsabilidades:
• Proveedor: Software y datos alojados
• Cliente: Uso del software y protección de datos personales

Ejemplos:

  • Propietarios: Google Workspace, Microsoft Office 365, Salesforce
  • Open Source: WordPress.com, Odoo, Nextcloud

🔧 Otros Modelos de Servicio

BpaaS: Business Process as a Service - Externalización de procesos de negocio
FaaS: Functions as a Service - Serverless computing (AWS Lambda, Azure Functions)
SECaaS: Security as a Service - Servicios de seguridad gestionada
BaaS: Backend as a Service - Herramientas para desarrollo móvil/web
DBaaS: Database as a Service - Bases de datos como servicio
NaaS: Network as a Service - VPN, ancho de banda bajo demanda

💼 Modelos de Negocio

🏭 Proveedor

Prestación directa de servicios cloud al cliente

🔗 Intermediario

Actúa entre cliente y proveedores (ej: AWS Marketplace)

🛠️ Habilitador

Venta de software/hardware para desarrollar servicios cloud

🔍 Auditor

Evaluaciones independientes de seguridad y cumplimiento

❓ Preguntas Frecuentes

¿Cuáles son las principales diferencias entre IaaS, PaaS y SaaS?

  • IaaS: Proporciona infraestructura (servidores, almacenamiento, redes). El cliente gestiona SO, aplicaciones y datos.
  • PaaS: Proporciona plataforma de desarrollo. El cliente solo gestiona aplicaciones y datos.
  • SaaS: Proporciona aplicaciones completas. El cliente solo usa el software.

¿Qué tipo de nube recomienda el ENS para sistemas de categoría alta?

Para sistemas de categoría alta, el ENS recomienda:

  • ✅ Nube privada interna o externalizada
  • ✅ Nube comunitaria interna
  • ❌ No se recomienda nube pública

¿Cuáles son los principales beneficios de GAIA-X?

GAIA-X busca crear una alternativa europea con:

  • Soberanía digital europea
  • Transparencia y apertura
  • Interoperabilidad (principios FAIR)
  • Independencia de proveedores no europeos
  • Innovación y evolución continua

¿Cómo se debe contratar servicios cloud en la administración pública?

Según la normativa vigente:

  • Los contratos cloud se tipifican como contratos de servicios (Ley 9/2017)
  • Se debe usar el acuerdo marco AM 27/2023 cuando esté disponible
  • Hasta entonces, licitación abierta
  • Contratos separados con proveedor e intermediario

¿Qué medidas de seguridad específicas exige el ENS para servicios cloud?

El RD 311/2022 incluye medidas específicas:

  • Conformidad con ENS o guías CCN-STIC
  • Auditorías de penetración (pentesting)
  • Transparencia en gestión de datos
  • Cifrado y gestión de claves adecuada
  • Definición de jurisdicción de datos
