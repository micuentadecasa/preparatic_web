📈 Resumen Ejecutivo

La Ciencia de Datos es un campo académico interdisciplinario que utiliza estadísticas, computación científica, métodos, procesos, algoritmos y sistemas científicos para extraer conocimientos de datos estructurados y no estructurados.

Pioneros Históricos

1962 - John W. Tukey

Estadístico estadounidense que cuestionó el futuro de la estadística como ciencia empírica. Creador del diagrama de caja y bigotes.

1974 - Peter Naur

Científico danés, ganador del premio Turing (2005), quien acuñó el término "Ciencia de Datos".

2001 - William S. Cleveland

Introdujo la Ciencia de Datos como disciplina unificada e independiente de la Estadística tradicional.

🔍 Tipos de Análisis

Análisis Descriptivo

Examina datos para obtener información sobre lo que ha ocurrido. Utiliza visualizaciones como gráficos y tablas.

Análisis Diagnóstico

Examen profundo para entender por qué ocurrió algo. Usa minería de datos y correlaciones.

Análisis Predictivo

Utiliza datos históricos para predecir patrones futuros mediante machine learning.

Análisis Prescriptivo

Predice y sugiere respuestas óptimas usando simulación y redes neuronales.

🔄 Ciclo Completo de Modelado de Datos

1. Obtención de Datos

Los datos masivos vienen de múltiples fuentes, diversos volúmenes, se generan rápidamente y deben verificarse su corrección.

2. Preprocesamiento

Tratamiento inicial: limpieza y filtrado de datos que no cumplen criterios de calidad o contienen errores.

3. Transformación e Integración

Homogenización de datos de múltiples fuentes para hacerlos comparables (estructurados vs no estructurados).

4. Análisis de Datos

Procesamiento usando algoritmos y métodos estadísticos para responder preguntas de los científicos de datos.

5. Interpretación

Evaluación de resultados aplicando experiencia del campo para entender, completar y corregir la información.

6. Validación

Verificación de robustez y detección de sesgos mediante técnicas externas o diferentes enfoques.

📥 ETL (Extract, Transform, Load)

Extracción

Transformación

Limpieza, depuración, corrección y estandarización de datos

Carga

Incorporación a estructuras OLAP, sobrescribiendo o agregando información

🧹 Preprocesado de Datos

Pasos del Preprocesado

  1. Recopilar/Crear datos: Set de datos disponible
  2. Descubrimiento: Búsqueda de conjuntos importantes
  3. Limpiar y validar: Data Scrubbing/Cleansing
  4. Reducción: Selección de datos relevantes
  5. Enriquecer: Agregación y conexión de datos
  6. Almacenar: Preparación para uso futuro

Técnicas de Reducción

  • Selección de características e instancias
  • Data squashing (datasets más pequeños)
  • Discretización

Herramientas Principales

  • Pandas y Numpy (Python)
  • Weka, RapidMiner, R Studio
  • Azure ML, IBM Watson Studio

🎯 Modelado de Datos

Objetivo: producir datos de mayor calidad, estructurados y consistentes para aplicaciones empresariales.

Modelo Conceptual

Orden teórico sobre datos según entidades del mundo real

Modelo Lógico

Establece valores clave, entidades discretas y relaciones en estructura lógica

Modelo Físico

Divide datos en tablas, índices y clústeres reales necesarios para almacenamiento

Algoritmos de Modelado

  • Árboles de Decisión: Clasificación mediante división lógica del dataset
  • Redes Neuronales: Reconocimiento de patrones complejos
  • SVM: Algoritmos supervisados para clasificación/regresión
  • Regresión Logística/Lineal: Predicción de variables categóricas/numéricas
  • Series Temporales: Combinación de técnicas tradicionales y forecasting

📊 Fundamentos Estadísticos

La estadística juega un papel clave en la creación de modelos predictivos a partir de datos históricos.

📈 Estadística Descriptiva

Técnica matemática que obtiene, organiza, presenta y describe conjuntos de datos.

Conceptos Fundamentales

  • Dispersión: Distancia entre valores de una variable
  • Promedio: Media aritmética de los valores
  • Sesgo/Curtosis: Cualidad de la curva de datos
  • Asimetría: Distribución de datos respecto al promedio

Tipos de Gráficos

  • Gráficos de barras, círculo, lineales
  • Polígonos, diagramas de dispersión
  • Histogramas y box plots

🔍 Estadística Inferencial

Determina propiedades de una población estadística a partir de una muestra mediante inducción.

Categorías Principales

  • Pruebas de hipótesis: Validación de conclusiones construidas
  • Intervalos de confianza: Estimación de parámetros poblacionales

Tipos

  • Paramétrica: Asume distribuciones específicas
  • No paramétrica: Sin asumir distribuciones

Características

Al ser resultado de cálculo probabilístico, conlleva cierto margen de error pero permite generalizar conclusiones.

⚖️ Métodos de Validación

Estrategias para estimar capacidad predictiva de modelos con nuevas observaciones.

Validación Simple

Reparto aleatorio en dos grupos: entrenamiento y evaluación

Leave One Out (LOOCV)

Método iterativo usando todas las observaciones excepto una

K-Fold Cross-Validation

División aleatoria en k grupos, k-1 para entrenar y 1 para validar

Bootstrapping

Muestreo aleatorio con reposición del mismo tamaño que muestra original

🤖 MLOps - Machine Learning Operations

Parte de la cultura DevOps relacionada con modelos de ML y automatización de procesos.

Objetivo

Desarrollar, entrenar y desplegar modelos en producción de manera fiable con procedimientos automatizados.

Tres Niveles de Implementación

  • Data: Datos, fases, ingestión, curado
  • Model: Testing, evaluación, empaquetado, despliegue
  • Code: Código donde se ejecuta el modelo

Beneficios de MLOps

  • Mejora comunicación entre equipos técnicos
  • Facilita y acelera gestión de datos
  • Genera flujos automáticos
  • Reduce sesgo en toma de decisiones

Herramientas Populares

  • Mlflow, DVC, Kubeflow, ONNX
  • TensorFlow Extended, Apache Airflow
  • Jupyter, PyTorch, Metaflow

📊 Big Data - Las 5 Vs

Según Gartner: "activos de información caracterizados por volumen elevado, velocidad elevada y alta variedad".

Volumen

Gran cantidad de datos a procesar

Velocidad

Datos en movimiento y creación en tiempo real

Variedad

Diferentes tipos de fuentes y datos

Veracidad

Grado de fiabilidad e incertidumbre

Valor

Capacidad de transformar datos en información útil

🐘 Ecosistema Hadoop

Inspirado en Google File System (GFS) y paradigma MapReduce para procesamiento distribuido.

Componentes Principales

HDFS

Sistema de archivos distribuido optimizado para grandes cantidades de datos con alta disponibilidad y tolerancia a fallos.

MapReduce

Paradigma de programación para escalabilidad masiva. Map divide datos en tuplas clave/valor, Reduce combina resultados.

YARN

Gestor de recursos que permite ejecutar múltiples aplicaciones compartiendo gestión común de recursos.

🔧 Herramientas del Ecosistema Hadoop

Captura y Manipulación

Sqoop

Transferencia masiva entre Hadoop y sistemas relacionales

Flume

Captura y movimiento de grandes volúmenes de logs

Kafka

Plataforma distribuida de streaming en tiempo real

Chukwa

Recolección, análisis y visualización de logs

Almacenamiento

Hive

Data Warehouse con lenguaje HiveQL similar a SQL

HBase

Base de datos NoSQL orientada a columnas

Impala

Motor SQL de baja latencia para análisis interactivo

Procesamiento

Spark

Framework de computación en clúster, 100x más rápido en memoria que MapReduce

Storm

Procesamiento en tiempo real con topologías master-slave

Pig

Lenguaje PigLatin para análisis de datos, traduce a MapReduce

🗄️ Bases de Datos NoSQL

Alternativa a bases relacionales con características específicas para Big Data.

Características

  • No utilizan SQL como lenguaje principal
  • No usan estructuras fijas como tablas
  • No permiten operaciones JOIN
  • Emplean arquitecturas distribuidas
  • Alta escalabilidad horizontal

Tipos Principales

Clave-Valor

Cada elemento con clave única

Ej: Redis, DynamoDB

Documentos

Datos semi-estructurados en XML/JSON

Ej: MongoDB, CouchDB

Grafos

Nodos y aristas para relaciones complejas

Ej: Neo4j, OrientDB

Columnas

Enfoque columnar vs transaccional

Ej: Cassandra, HBase

📚 Glosario de Términos Clave

Big Data

Activos de información caracterizados por volumen elevado, velocidad elevada y alta variedad que demandan soluciones innovadoras de procesado.

CRISP-DM

Cross-Industry Standard Process for Data Mining. Metodología estándar para proyectos de minería de datos con 6 fases principales.

Data Lake

Repositorio de almacenamiento que contiene grandes cantidades de datos en su formato nativo hasta que se necesiten.

Data Warehouse

Sistema de almacenamiento de datos diseñado para consultas y análisis en lugar de procesamiento de transacciones.

ETL

Extract, Transform, Load. Proceso que permite mover datos de diferentes fuentes, formatearlos y cargarlos en un destino.

HDFS

Hadoop Distributed File System. Sistema de archivos distribuido optimizado para almacenar grandes volúmenes de datos.

Machine Learning

Subcampo de la inteligencia artificial que permite a las computadoras aprender sin ser explícitamente programadas.

MapReduce

Paradigma de programación para procesamiento paralelo de grandes conjuntos de datos en clusters distribuidos.

MLOps

Machine Learning Operations. Conjunto de prácticas para automatizar y simplificar flujos de trabajo de ML en producción.

NoSQL

Bases de datos no relacionales que no utilizan SQL tradicional, diseñadas para escalabilidad horizontal y flexibilidad.

OLAP

Online Analytical Processing. Tecnología para análisis multidimensional rápido de datos empresariales.

Web Scraping

Técnica para extraer información de sitios web de forma automatizada mediante software especializado.

❓ Preguntas Frecuentes

¿Cuál es la diferencia entre Data Mining y Machine Learning?

Data Mining se centra en descubrir patrones en datos existentes, mientras que Machine Learning construye modelos que pueden predecir o clasificar nuevos datos basándose en patrones aprendidos.

¿Cuándo usar bases de datos NoSQL vs SQL?

NoSQL es mejor para datos no estructurados, escalabilidad horizontal y desarrollo ágil. SQL es mejor para datos estructurados, transacciones ACID y consultas complejas.

¿Qué es mejor: Python o R para ciencia de datos?

Python es mejor para desarrollo integral de aplicaciones y producción. R es superior para análisis estadístico especializado y investigación académica.

¿Cómo elegir el algoritmo de ML apropiado?

Depende del tipo de problema (clasificación, regresión, clustering), tamaño de datos, interpretabilidad requerida y recursos computacionales disponibles.

¿Qué roles son esenciales en un equipo de Data Science?

Data Scientist, Data Engineer, Data Analyst, ML Engineer y Data Steward son roles fundamentales, cada uno con responsabilidades específicas en el pipeline de datos.
