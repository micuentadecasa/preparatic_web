Algoritmos de Cifrado

Tema 079: Simétricos, Asimétricos, Función Hash y Notariado Digital

Resumen General del Tema

🎯 Objetivo Principal

Comprender los fundamentos de la criptografía moderna, incluyendo algoritmos simétricos, asimétricos, funciones hash y esquemas híbridos para garantizar la seguridad de la información.

📊 Estadísticas Clave

  • AES: Estándar actual con claves de 128/192/256 bits
  • RSA: Claves típicas de 2048-4096 bits
  • SHA-2: Hash recomendado con 256-512 bits
  • Velocidad: Simétrico 100x más rápido que asimétrico

🔑 Conceptos Fundamentales

  • Confidencialidad: Solo el destinatario lee
  • Integridad: Detectar modificaciones
  • Autenticidad: Verificar origen
  • No repudio: Prueba irrefutable

  • Distinguir entre cifrado simétrico y asimétrico
  • Identificar algoritmos recomendados y obsoletos
  • Comprender las propiedades de las funciones hash
  • Aplicar esquemas híbridos en casos prácticos
  • Evaluar la seguridad de sistemas criptográficos

Algoritmos de Cifrado Simétrico

🔐 Característica Principal

Una única clave (KA = KB) para cifrar y descifrar. La clave debe ser secreta y conocida solo por emisor y receptor.

🌊 Cifradores de Flujo

Combinan dígito a dígito el flujo de datos con un keystream generado pseudo-aleatoriamente.

Perfiles de Uso:

  • Aplicaciones críticas: Requisitos estrictos de velocidad
  • Hardware limitado: Capacidad limitada de cómputo y almacenamiento
Algoritmo Estado Uso Histórico Observaciones
A5/1 y A5/2 NO GSM Obsoleto, vulnerabilidades conocidas
RC4 NO WEP, WPA, SSL Vulnerabilidades críticas
E0 NO Bluetooth Reemplazado por algoritmos más seguros
Rabbit SÍ Actual Clave de 128 bits
SNOW3G SÍ UMTS Clave de 128 bits, telecomunicaciones

🧱 Cifradores de Bloque

El texto se divide en bloques de tamaño fijo para su cifrado.

Modos de Operación:

ECB (Electronic Code Book): Bloques cifrados independientemente
CBC (Cipher Block Chaining): Cifrado depende del bloque anterior
CTR (Contador): Usa cifrador de bloque para generar keystream
OFB (Output Feedback): Realimentación de salida
CFB (Cipher Feedback): Realimentación de cifrador
Algoritmo Estado Características
DES NO 56 bits + 8 paridad, Feistel
TripleDES NO 112 bits, DES en cascada
Blowfish NO 32-448 bits
AES SÍ 128/192/256 bits, Rijndael
Camellia SÍ 128/192/256 bits

Algoritmos de Cifrado Asimétrico

🔑 Característica Principal

Dos claves diferentes pero complementarias: una pública y una privada (KA ≠ KB)

🔒 Cifrado (Confidencialidad)

Emisor: Cifra con clave pública del receptor

Receptor: Descifra con su clave privada

✅ Solo el destinatario puede leer

✍️ Firma (Autenticidad)

Emisor: Cifra con su clave privada

Receptor: Descifra con clave pública del emisor

✅ Garantiza origen y no repudio

📊 Clasificación por Problemas Matemáticos

🧮 Factorización Entera

RSA: El más utilizado, basado en la dificultad de factorizar números grandes

📈 Logaritmo Discreto

Diffie-Hellman: Intercambio de claves

DSA: Digital Signature Algorithm

ElGamal: Cifrado y firma

🔄 Logaritmo Discreto Elíptico

ECDH: Elliptic Curve Diffie-Hellman

ECDSA: Elliptic Curve DSA

Ventaja: Claves más cortas con igual seguridad

⚖️ Ventajas vs Desventajas

✅ Ventajas

  • No repudio
  • Distribución de claves
  • Escalabilidad
  • Autenticación

❌ Desventajas

  • Alto coste computacional
  • Claves más largas
  • Velocidad más lenta
  • Gestión compleja

Función Hash

🎯 Definición

Funciones que transforman un mensaje de cualquier longitud en una salida de longitud fija (hash o digest)

🔍 Propiedades Fundamentales

📏 Compresión

La salida es de longitud fija e independiente de la longitud del mensaje

🌊 Difusión

La salida es aleatoria y función de todos los bits de entrada

🔒 Resistencia a la Preimagen

Dado un hash, es computacionalmente imposible obtener el mensaje original

⚡ Resistencia Débil a Colisión

Dado m1 y h(m1), imposible encontrar m2 tal que h(m1) = h(m2)

💪 Resistencia Fuerte a Colisión

Imposible encontrar dos mensajes diferentes con el mismo hash

🕵️ MDC (Message Detection Code)

Detección de modificaciones en el mensaje

Uso: h(mensaje) = hash

Verifica: Integridad

🛡️ MAC (Message Authentication Code)

Detección + autenticación del origen

Uso: h(mensaje + clave) = MAC

Verifica: Integridad + Autenticidad

Esquemas Híbridos

💡 Concepto

Combinan lo mejor de ambos mundos: velocidad del simétrico + seguridad del asimétrico

📧 Sobre Digital (Confidencialidad)

🚀 Proceso del Emisor:

1. Genera una clave simétrica aleatoria
2. Cifra el mensaje con algoritmo simétrico y la clave generada
3. Cifra la clave simétrica con algoritmo asimétrico y clave pública del receptor
4. Envía: mensaje cifrado + clave cifrada

📥 Proceso del Receptor:

1. Descifra la clave simétrica con su clave privada (algoritmo asimétrico)
2. Descifra el mensaje con la clave simétrica (algoritmo simétrico)
💡 Ventaja Principal: Especialmente útil en envíos multidestino, permitiendo cifrar el mensaje una única vez

✍️ Firma Digital (Autenticidad + Integridad + No Repudio)

📝 Proceso del Emisor:

1. Calcula el hash del mensaje
2. Cifra el hash con algoritmo asimétrico y su clave privada
3. Envía: mensaje + hash cifrado (firma)

🔍 Proceso del Receptor:

1. Calcula el hash del mensaje recibido
2. Descifra el hash firmado con la clave pública del emisor
3. Compara ambos hashes: si coinciden, la firma es válida

🔄 Comparación Simétrico vs Asimétrico

Aspecto Simétrico Asimétrico
Velocidad 🟢 Rápido 🔴 Lento
Longitud clave 🟢 Corta 🔴 Larga
No repudio 🔴 No 🟢 Sí
Distribución 🔴 Compleja 🟢 Simple

❓ Preguntas Frecuentes

¿Por qué DES se considera inseguro?
DES utiliza claves de solo 56 bits, lo que permite ataques de fuerza bruta con hardware moderno en tiempo razonable. Fue diseñado en los años 70 cuando la potencia computacional era limitada.
¿Cuál es la diferencia entre SHA-2 y SHA-3?
SHA-2 y SHA-3 son diferentes familias de funciones hash. SHA-3 utiliza la construcción Keccak, mientras que SHA-2 usa Merkle-Damgård. Ambos son seguros, pero SHA-3 ofrece mayor flexibilidad y resistencia a ciertos tipos de ataques.
¿Qué longitud de clave RSA se recomienda actualmente?
Se recomiendan claves RSA de al menos 2048 bits para uso general, y 3072-4096 bits para información muy sensible o uso a largo plazo. Claves de 1024 bits se consideran inseguras.
¿Por qué las curvas elípticas son más eficientes?
Las curvas elípticas ofrecen el mismo nivel de seguridad con claves más cortas. Por ejemplo, una clave ECC de 256 bits equivale aproximadamente a una RSA de 3072 bits, resultando en menor uso de recursos.
¿Cuándo usar cifrado de flujo vs cifrado de bloque?
Cifrado de flujo: cuando se necesita velocidad extrema o en dispositivos con recursos limitados. Cifrado de bloque: para la mayoría de aplicaciones generales, especialmente cuando se requiere procesamiento de grandes volúmenes de datos.
¿Qué es la criptoagilidad?
Es la capacidad de cambiar rápidamente algoritmos criptográficos en un sistema cuando se descubren vulnerabilidades o aparecen nuevas amenazas, sin afectar significativamente la funcionalidad del sistema.

📖 Glossario de Términos Clave

Algoritmo: Conjunto de reglas matemáticas para cifrar/descifrar
Clave: Información secreta usada por el algoritmo
Cifrado: Proceso de convertir texto plano en texto cifrado
Descifrado: Proceso inverso al cifrado
Hash: Función que produce una salida de longitud fija
Keystream: Secuencia de bits pseudo-aleatoria
Nonce: Número usado solo una vez
Padding: Relleno para completar bloques
Salt: Valor aleatorio añadido antes del hash
Vector de Inicialización (IV): Valor inicial aleatorio

⚡ Mejores Prácticas

  • Usar solo algoritmos criptográficos recomendados por organismos oficiales
  • Generar claves con suficiente entropía usando generadores seguros
  • Implementar rotación periódica de claves
  • Nunca reutilizar nonces o vectores de inicialización
  • Validar y actualizar regularmente bibliotecas criptográficas
  • Usar esquemas híbridos para combinar eficiencia y seguridad
  • Implementar gestión segura del ciclo de vida de las claves
  • Documentar y auditar implementaciones criptográficas
