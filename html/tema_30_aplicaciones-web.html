Desarrollo Frontend

🎯 Frontend: La Interfaz de Usuario

El desarrollo frontend se centra en la experiencia del usuario, incluyendo diseño responsivo, interactividad, accesibilidad y rendimiento.

HTML5 Semántico

  • Elementos semánticos: header, nav, main, article
  • APIs nativas: Canvas, Web Storage, Geolocation
  • Formularios avanzados con validación
  • Multimedia: audio, video, elementos interactivos
  • Web Components y Custom Elements

CSS3 Avanzado

  • Flexbox y CSS Grid para layouts
  • Animaciones y transiciones CSS
  • Variables CSS (Custom Properties)
  • Media Queries para diseño responsivo
  • Metodologías: BEM, SMACSS, Atomic CSS

JavaScript Moderno

  • ES6+: Arrow functions, destructuring, modules
  • Programación asíncrona: Promises, async/await
  • DOM manipulation y Event handling
  • Fetch API y XMLHttpRequest
  • TypeScript para tipado estático

Frameworks y Librerías

  • React: Virtual DOM, JSX, Hooks
  • Vue.js: Directivas, Vuex, Vue Router
  • Angular: Components, Services, Dependency Injection
  • Estado global: Redux, Vuex, NgRx
  • UI Libraries: Bootstrap, Material-UI, Tailwind

Servicios Web y APIs

🔗 Interoperabilidad y Comunicación

Los servicios web permiten la comunicación entre aplicaciones distribuidas usando estándares como REST, SOAP y GraphQL.

REST APIs

  • Principios REST: Stateless, Cacheable, Uniform Interface
  • Métodos HTTP: GET, POST, PUT, DELETE, PATCH
  • Códigos de estado HTTP y manejo de errores
  • Versionado de APIs y documentación
  • HATEOAS y Richardson Maturity Model

GraphQL

  • Schema y Type System
  • Queries, Mutations y Subscriptions
  • Resolvers y Data Sources
  • Apollo Server y Apollo Client
  • Federación de esquemas

SOAP y Servicios Web

  • WSDL (Web Services Description Language)
  • UDDI (Universal Description, Discovery, Integration)
  • WS-Security y WS-Addressing
  • Envelope, Header y Body
  • Binding protocols: HTTP, SMTP, TCP

WebSockets y Tiempo Real

  • Comunicación bidireccional full-duplex
  • Socket.IO para Node.js
  • Server-Sent Events (SSE)
  • WebRTC para comunicación P2P
  • SignalR para aplicaciones .NET

Ejercicios de Aplicaciones Web

Progreso: 0/10

Ejercicio 1: Cálculo de Throughput del Servidor

Un servidor web procesa 2400 requests en 4 minutos. ¿Cuál es el throughput en requests por segundo?

Throughput = Total Requests / Tiempo (segundos)

Ejercicio 2: Tiempo de Carga de Recursos

Una página web tiene 8 archivos CSS (50KB cada uno), 12 imágenes (200KB cada una) y 6 archivos JS (100KB cada uno). ¿Cuál es el tamaño total en MB?

Tamaño Total = Σ(Cantidad × Tamaño individual)

Ejercicio 3: Cálculo de Rate Limiting

Una API permite 1000 requests por hora por usuario. Un usuario ha hecho 750 requests en 45 minutos. ¿Cuántos requests puede hacer en los próximos 15 minutos?

Requests Disponibles = Límite - Requests Usados

Ejercicio 4: Optimización de Bundle JavaScript

Un bundle JS inicial pesa 2.5MB. Tras code splitting se divide en: chunk principal (800KB), chunk vendor (1.2MB) y 3 chunks lazy (200KB cada uno). ¿Cuál es la reducción del bundle inicial?

Bundle Inicial = Chunk Principal + Chunk Vendor

Ejercicio 5: Cálculo de CDN Savings

Un servidor maneja 10,000 requests/día de archivos estáticos (ancho de banda: 500MB/día). Con CDN se reduce el tráfico en 80%. Si el coste por GB es $0.10, ¿cuál es el ahorro mensual?

Ahorro = Ancho Banda × % Reducción × Días × Coste/GB

Ejercicio 6: Performance Budget

Una página tiene un presupuesto de 2 segundos para FCP. Los archivos tardan: HTML (200ms), CSS (300ms), JS crítico (800ms), fuentes (400ms). ¿Se cumple el presupuesto?

FCP = max(HTML + CSS, HTML + JS crítico, HTML + Fuentes)

Ejercicio 7: Conexiones HTTP/2

Una página necesita cargar 20 recursos. Con HTTP/1.1 (máx 6 conexiones paralelas) vs HTTP/2 (multiplexing). Si cada recurso tarda 100ms, ¿cuál es la mejora de tiempo con HTTP/2?

HTTP/1.1 = ⌈Recursos/Conexiones⌉ × Tiempo por recurso
HTTP/2 = Tiempo por recurso (paralelo completo)

Ejercicio 8: Base de Datos - Pool de Conexiones

Un pool tiene 10 conexiones BD. Cada query tarda 50ms. Si llegan 25 requests simultáneos, ¿cuál es el tiempo total para procesar todas?

Tiempo Total = ⌈Requests/Pool Size⌉ × Tiempo por Query

Ejercicio 9: API Caching - Hit Rate

Una API recibe 1000 requests. El cache tiene 750 hits y 250 misses. Cada hit tarda 10ms y cada miss 80ms. ¿Cuál es el tiempo promedio de respuesta?

Tiempo Promedio = (Hits × T_hit + Misses × T_miss) / Total

Ejercicio 10: WebSocket vs HTTP Overhead

Una aplicación envía 100 mensajes de 50 bytes cada uno. HTTP añade 500 bytes de headers por request. WebSocket añade solo 2 bytes por mensaje tras la conexión inicial. ¿Cuál es la reducción de overhead?

Overhead HTTP = Mensajes × Headers por mensaje
Overhead WS = Mensajes × Headers por mensaje WS
