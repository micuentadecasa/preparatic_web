🌍 Evolución Histórica de Internet

1969 - ARPANET

Proyecto DARPA para red resistente a fallos. Primeros 4 nodos: UCLA, Stanford, UCSB, Utah. Protocolo NCP (Network Control Program).

1983 - TCP/IP

Adopción del protocolo TCP/IP. Separación ARPANET militar y civil. Nacimiento de la Internet moderna.

1986 - NSFNET

National Science Foundation crea backbone académico. Velocidad inicial 56 Kbps, evolución a T3 (45 Mbps).

1991 - Comercialización

Eliminación restricciones comerciales. WWW de Tim Berners-Lee. Primer navegador web Mosaic (1993).

1995-2000 - Boom Internet

ISPs comerciales, dot-com boom. Internet2 para investigación. IPv6 desarrollo (RFC 2460, 1998).

Tier Características Función Ejemplos

Tier 2 Red regional, paga tránsito a Tier 1 ISPs regionales grandes Telefónica, Orange, Comcast
Tier 3 Red local, compra todo el tránsito ISPs locales y empresas ISPs locales, empresas

🔗 Puntos de Intercambio y Peering

Internet Exchange Points (IXPs)

  • Función: Intercambio tráfico entre ISPs sin costes de tránsito
  • Ventajas: Menor latencia, mayor control rutas, reducción costes
  • Tipos: Público (neutro) vs privado (bilateral)
  • Ejemplos: DE-CIX (Frankfurt), AMS-IX (Ámsterdam), ESPANIX (Madrid)

💡 Pregunta de Examen Típica

¿Cuál es la diferencia entre peering y tránsito?

Respuesta: Peering: intercambio gratuito entre iguales. Tránsito: pago por acceso a rutas globales.

🧠 Web Semántica (Web 3.0)

Evolución hacia una web donde las máquinas pueden entender e interpretar el contenido para proporcionar experiencias más inteligentes.

Tecnologías Clave

  • RDF (Resource Description Framework): Modelo datos para recursos web
  • OWL (Web Ontology Language): Vocabularios y relaciones semánticas
  • SPARQL: Lenguaje consulta para datos RDF
  • Linked Data: Interconexión de datos estructurados
  • JSON-LD: Formato ligero para linked data

📊 Evolución de la Web

🌐 Internet of Things (IoT)

Ecosistema de objetos físicos conectados que recopilan e intercambian datos mediante sensores, software y conectividad de red.

Capa de Dispositivos

Sensores, actuadores, microcontroladores. Protocolos: MQTT, CoAP, LoRaWAN. Restricciones: energía, procesamiento, memoria.

Capa de Conectividad

Redes: WiFi, Bluetooth, Zigbee, 5G, NB-IoT. Gateways para protocolo translation y agregación datos.

Capa de Datos

Plataformas IoT: AWS IoT, Azure IoT, Google Cloud IoT. Analytics en tiempo real, machine learning.

Capa de Aplicación

Smart homes, ciudades inteligentes, industria 4.0, agricultura de precisión, wearables de salud.

⛓️ Blockchain - Fundamentos

Tecnología de registro distribuido que mantiene una lista continuamente creciente de registros, llamados bloques, unidos mediante criptografía.

Características Principales

  • Descentralización: No autoridad central, red P2P
  • Inmutabilidad: Registros no pueden modificarse retroactivamente
  • Transparencia: Todas las transacciones son públicas
  • Consenso: Acuerdo distribuido sobre estado del sistema
  • Criptografía: Hash SHA-256, firmas digitales

🔗 Estructura de Bloques

Componentes de un Bloque

  • Block Header: Metadatos del bloque
  • Previous Block Hash: Enlace al bloque anterior
  • Merkle Root: Resumen criptográfico de transacciones
  • Timestamp: Marca temporal de creación
  • Nonce: Número usado una vez para prueba de trabajo
  • Transactions: Lista de transacciones validadas

🤝 Mecanismos de Consenso

Mecanismo Funcionamiento Ventajas Desventajas
Proof of Work Resolver puzzle criptográfico Muy seguro, probado Alto consumo energético
Proof of Stake Validadores según stake Eficiente energéticamente Riesgo centralización
Practical BFT Consenso inmediato Rápido, final Limitado participantes
Proof of Authority Validadores pre-aprobados Rápido, eficiente Centralizado

💡 Pregunta de Examen Típica

¿Cuál es la diferencia entre blockchain público y privado?

Respuesta: Público: abierto a todos, descentralizado. Privado: acceso restringido, mayor control.
