Administración de Sistemas Operativos

🖥️ Gestión Integral del Sistema Operativo

La administración de sistemas operativos abarca desde la instalación inicial hasta el mantenimiento continuo, incluyendo configuración, seguridad, rendimiento y disponibilidad.

Gestión de Usuarios y Permisos

  • Creación y eliminación de cuentas
  • Configuración de grupos y roles
  • Políticas de contraseñas
  • Control de acceso (ACL)
  • Auditoría de accesos
  • Autenticación centralizada (LDAP, AD)

Configuración del Sistema

  • Parámetros del kernel
  • Variables de entorno
  • Servicios y demonios
  • Configuración de red
  • Firewall y seguridad
  • Programación de tareas (cron, Task Scheduler)

Gestión de Software

  • Instalación de paquetes
  • Actualizaciones y parches
  • Gestión de dependencias
  • Repositorios de software
  • Licenciamiento
  • Rollback y versionado

Monitoreo y Rendimiento

  • Métricas de sistema (CPU, RAM, I/O)
  • Análisis de logs
  • Alertas y notificaciones
  • Análisis de tendencias
  • Optimización de rendimiento
  • Planificación de capacidad

Instalación Inicial

Particionado, instalación del SO, configuración básica de red, usuarios iniciales y servicios esenciales.

Configuración de Seguridad

Hardening del sistema, configuración de firewall, políticas de acceso y auditoría.

Implementación de Monitoreo

Instalación de agentes, configuración de métricas, establecimiento de umbrales y alertas.

Automatización

Scripts de mantenimiento, backup automático, actualización de parches y tareas rutinarias.

# Comandos básicos de administración Linux sudo useradd -m -s /bin/bash usuario sudo usermod -aG sudo usuario sudo systemctl enable servicio sudo systemctl start servicio sudo crontab -e df -h | head -10 top -bn1 | grep "Cpu(s)" free -h tail -f /var/log/syslog

Métricas de Rendimiento del Sistema

CPU Usage = (Total Time - Idle Time) / Total Time × 100
Memory Usage = Used Memory / Total Memory × 100
Disk I/O = (Read + Write operations) / Time interval

Métricas de Performance de BD

Query Response Time = Parse + Execute + Fetch time
Throughput = Transactions / Time interval
Buffer Hit Ratio = (1 - Physical Reads / Logical Reads) × 100

Monitoreo y Gestión de Rendimiento

📊 Monitoreo Proactivo

El monitoreo continuo permite detectar problemas antes de que afecten a los usuarios, optimizar el rendimiento y planificar el crecimiento de la infraestructura.

Métricas del Sistema

  • CPU: utilización, load average, procesos
  • Memoria: RAM utilizada, swap, buffers
  • Disco: espacio libre, I/O, latencia
  • Red: throughput, paquetes, errores
  • Temperatura y voltajes
  • Servicios y procesos activos

Métricas de Base de Datos

  • Conexiones activas y máximas
  • Transacciones por segundo
  • Tiempo de respuesta de queries
  • Cache hit ratio
  • Locks y deadlocks
  • Tamaño de logs y backups

Herramientas de Monitoreo

  • Nagios: monitoreo de infraestructura
  • Zabbix: monitoreo integral
  • Prometheus + Grafana: métricas y visualización
  • ELK Stack: análisis de logs
  • New Relic: APM y monitoreo
  • PRTG: monitoreo de red

Alertas y Notifications

  • Umbrales críticos y warning
  • Escalado de alertas
  • Notificaciones por email, SMS, Slack
  • Supresión de alertas duplicadas
  • Documentación de incidentes
  • Métricas de disponibilidad (SLA)

# Scripts de monitoreo básico #!/bin/bash # Monitoreo de recursos del sistema echo "=== System Resources Monitor ===" echo "CPU Usage: $(top -bn1 | grep "Cpu(s)" | awk '{print $2}' | cut -d'%' -f1)" echo "Memory Usage: $(free | grep Mem | awk '{printf "%.1f%%", $3/$2 * 100.0}')" echo "Disk Usage: $(df -h / | awk 'NR==2{printf "%s", $5}')" echo "Load Average: $(uptime | awk -F'load average:' '{print $2}')" echo "Active Connections: $(netstat -an | grep ESTABLISHED | wc -l)"

Ejercicios de Administración de Sistemas

Progreso: 0/10

Ejercicio 1: Cálculo de Utilización de CPU

Un servidor registra los siguientes tiempos en 100 segundos: user=20s, system=15s, idle=60s, wait=5s. ¿Cuál es el porcentaje de utilización de CPU?

CPU Usage = (Total - Idle) / Total × 100

Ejercicio 2: Planificación de Backup

Una base de datos de 500GB crece 50GB/semana. El backup full tarda 4 horas y el incremental 30 minutos. ¿Cuántas horas de backup por semana con estrategia semanal full + diarios incrementales?

Tiempo Total = Full backup + (6 × Incremental backups)

Ejercicio 3: Buffer Hit Ratio

Una base de datos procesa 10,000 lecturas lógicas y 500 lecturas físicas de disco. ¿Cuál es el buffer hit ratio?

Buffer Hit Ratio = (1 - Physical Reads / Logical Reads) × 100

Ejercicio 4: Cálculo de MTTR

En un mes de 720 horas, un sistema tuvo 6 incidentes con downtimes de: 15min, 30min, 45min, 20min, 10min, 40min. ¿Cuál es el MTTR en minutos?

MTTR = Total Downtime / Number of Incidents

Ejercicio 5: Capacidad de Almacenamiento

Un sistema tiene 3 discos RAID 5: 2TB, 2TB, 2TB. ¿Cuál es la capacidad útil disponible después de la redundancia?

RAID 5 Capacity = (n-1) × Smallest Disk Size

Ejercicio 6: Monitoreo de Memoria

Un servidor tiene 32GB RAM total. Actualmente usa: Sistema=8GB, Aplicaciones=18GB, Cache=4GB, Libre=2GB. ¿Cuál es el porcentaje de memoria activamente utilizada?

Active Memory = (Sistema + Aplicaciones) / Total × 100

Ejercicio 7: SLA y Downtime Permitido

Un SLA garantiza 99.9% de uptime anual. ¿Cuántas horas de downtime están permitidas en un año de 8760 horas?

Downtime Permitido = Total Hours × (1 - SLA%)

Ejercicio 8: Throughput de Base de Datos

Una BD procesa 1200 transacciones en 5 minutos durante pico de carga. ¿Cuál es el throughput en TPS (transacciones por segundo)?

TPS = Total Transactions / Time in seconds

Ejercicio 9: Espacio de Logs de Base de Datos

Los logs de transacciones crecen 2GB/día. Se mantienen 7 días antes del truncate. Si el espacio total de logs es 20GB, ¿hay suficiente espacio?

Espacio Necesario = Crecimiento diario × Días de retención

Ejercicio 10: Eficiencia de Compresión de Backup

Un backup sin comprimir ocupa 800GB. Con compresión se reduce a 320GB. ¿Cuál es el porcentaje de reducción de espacio?

% Reducción = (Tamaño Original - Tamaño Comprimido) / Tamaño Original × 100
