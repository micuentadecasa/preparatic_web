🔄 Tipos de Conmutación

Los métodos de conmutación determinan cómo se establece la comunicación y se transfieren los datos entre origen y destino.

Tipo Características Ventajas Desventajas Ejemplos
Circuitos Camino dedicado durante comunicación Latencia constante, QoS garantizada Ineficiente, caro PSTN, ISDN
Paquetes Datos divididos en paquetes Eficiente, flexible, escalable Latencia variable IP, Internet
Celdas Paquetes de tamaño fijo (53 bytes) Latencia predecible, QoS Overhead, complejidad ATM

📡 Redes de Difusión vs Conmutadas

Redes de Difusión (Broadcast)

Un solo canal compartido por todos los nodos. Ejemplos: Ethernet clásico (bus), WiFi. Problema: colisiones y escalabilidad limitada.

Redes Conmutadas (Switched)

Múltiples enlaces punto a punto conectados por switches. Ventajas: sin colisiones, full-duplex, microsegmentación, escalabilidad.

Evolución Tecnológica

Migración de hubs (difusión) a switches (conmutación) para mejor rendimiento y seguridad en redes LAN modernas.

🔗 Integración Voz-Datos

Convergencia de Servicios

  • VoIP (Voice over IP): Digitalización de voz sobre redes IP
  • ToIP (Telephony over IP): Sistemas telefónicos completos sobre IP
  • Unified Communications: Integración voz, video, mensajería, presencia
  • Requisitos QoS: Latencia <150ms, jitter <30ms, pérdidas <1%

💡 Pregunta de Examen Típica

¿Cuáles son los requisitos de QoS para VoIP?

Respuesta: Latencia máxima 150ms, jitter menor 30ms, pérdida de paquetes inferior al 1%.

🌐 Ethernet Conmutada

Evolución de Ethernet desde redes compartidas (hubs) a redes conmutadas (switches) para eliminar dominios de colisión.

Características Principales

  • Microsegmentación: Cada puerto = dominio de colisión independiente
  • Full-Duplex: Transmisión simultánea en ambas direcciones
  • Auto-negotiation: Velocidad y duplex automáticos
  • Store-and-forward: Verificación de integridad de tramas
  • VLAN 802.1Q: Segmentación lógica, hasta 4094 VLANs

🏷️ MPLS (Multiprotocol Label Switching)

Componente Función Características
Labels Identificadores de 20 bits Conmutación rápida, sin lookup IP
LSR Label Switch Router Conmuta basado en labels
LSP Label Switched Path Camino predeterminado para tráfico
LDP Label Distribution Protocol Distribución de labels entre LSRs
Traffic Engineering Control explícito de rutas Optimización de ancho de banda

🔧 Tecnologías de Agregación

Link Aggregation (802.3ad / LACP)

Objetivo: Aumentar ancho de banda y redundancia

Métodos: Static LAG, LACP (Link Aggregation Control Protocol)

Balanceo: Por MAC, IP, puerto, hash algorithms

Capacidad: Hasta 8 enlaces físicos por grupo lógico

🗺️ Protocolos de Encaminamiento

Algoritmos que permiten a los routers construir y mantener tablas de enrutamiento para determinar el mejor camino hacia destinos.

Protocolo Tipo Métrica Convergencia Uso
RIP v1/v2 Distance Vector Hop count (máx 15) Lenta (minutos) Redes pequeñas
OSPF Link State Coste (ancho banda) Rápida (segundos) Redes corporativas
EIGRP Hybrid Compuesta (BW, delay) Muy rápida Cisco (propietario)
BGP Path Vector Políticas AS-Path Lenta pero estable Internet (ISPs)

🌍 OSPF - Características Avanzadas

Áreas Jerárquicas

Área 0 (backbone) + áreas normales. Reduce tabla de enrutamiento y tráfico LSA. ABRs conectan áreas.

Tipos de LSA

LSA Type 1 (Router), Type 2 (Network), Type 3 (Summary), Type 4 (ASBR Summary), Type 5 (External).

Elección DR/BDR

Designated Router en redes multi-acceso. Reduce adjacencias de n² a 2n. Prioridad configurable.

🌐 BGP - Border Gateway Protocol

Conceptos Clave BGP

  • AS (Autonomous System): Dominio administrativo único
  • eBGP: Entre diferentes AS (External BGP)
  • iBGP: Dentro del mismo AS (Internal BGP)
  • AS-Path: Lista de AS atravesados, previene loops
  • Atributos: Next-hop, Local-preference, MED, Community
  • Route Reflectors: Solución al full-mesh iBGP

⭐ Calidad de Servicio (QoS)

Conjunto de tecnologías que garantizan niveles predecibles de servicio para aplicaciones críticas mediante gestión de recursos de red.

Modelos de QoS

  • Best Effort: Sin garantías, tratamiento igual para todo tráfico
  • Integrated Services (IntServ): Reserva recursos por flujo (RSVP)
  • Differentiated Services (DiffServ): Clasificación por clases de tráfico

🏷️ Marcado y Clasificación QoS

Capa Campo Valores Descripción
Capa 2 CoS (802.1p) 0-7 (3 bits) Prioridad en trama Ethernet
Capa 3 ToS/DSCP 0-63 (6 bits) Marcado en cabecera IP
MPLS EXP bits 0-7 (3 bits) Experimental bits en label

🚦 Mecanismos de QoS

Classification & Marking

Identificación de tráfico (ACLs, protocolos) y marcado con valores CoS/DSCP para tratamiento diferenciado.

Policing & Shaping

Control de velocidad: Policing descarta exceso, Shaping bufferiza para suavizar ráfagas.

Queuing & Scheduling

Algoritmos: FIFO, PQ, WFQ, CBWFQ. Asignación de ancho de banda por cola de tráfico.

Congestion Avoidance

RED, WRED: Descarte proactivo antes de congestión total para evitar tail drops masivos.

📚 Errores Comunes Examen

  • Error: Confundir shaping con policing - Shaping bufferiza, policing descarta
  • Error: OSPF usa hop count - Usa coste basado en ancho de banda
  • Error: BGP es protocolo IGP - Es EGP para conexión entre AS
  • Error: MPLS trabaja en capa 3 - Opera entre capa 2 y 3 (2.5)
  • Error: VLANs aumentan dominios de colisión - Los reducen mediante segmentación
