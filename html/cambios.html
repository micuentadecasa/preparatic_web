🎯 Definición de Gestión de Cambios

Es la forma de dar respuesta a nuevas necesidades y modificaciones al software actual, suponiendo un proceso de mejora del software.

Motivaciones para los Cambios:

  • Modificación de requisitos y limitaciones iniciales
  • Errores operacionales o funcionales detectados
  • Adecuación del software a normativa legal
  • Necesidades de mejora o optimización

🏗️ ITIL 4 - Control de Cambios

Objetivo: Conseguir el máximo número de cambios exitosos en servicios, asegurando la gestión adecuada de los riesgos.

Componentes Clave:

  • Autoridad de cambio que analiza y valida
  • Gestión de planificación con recursos necesarios
  • Información a partes implicadas
  • Análisis de riesgos para cada cambio

🔄 Tipos de Cambios ITIL 4

⚡ Cambios Estándar

Bajo riesgo - Pre-autorizados, documentados y ejecutados sin necesidad de autorización adicional.

  • Operacionales
  • Peticiones rutinarias
📊 Cambios Normales

Planificados y analizados - Autorizados por el CAB (Comité Asesor del Cambio).

  • Iniciados por peticiones de cambio
  • Análisis completo de impacto
🚨 Cambios de Emergencia

Implementación urgente - Priorizados, autorizados por ECAB con análisis de riesgos.

  • Autorización más ágil
  • Autoridad diferente (ECAB)

📊 Definición de Gestión de Configuración

Conjunto de procesos destinados a asegurar la calidad de todo producto obtenido durante cualquiera de las etapas del desarrollo de un sistema de información.

Objetivos principales:

  • Estricto control de cambios realizados
  • Disponibilidad constante de versiones estables
  • Verificar componentes correctamente identificados
  • Control de almacenamiento y versionado

📋 Definición de Versión según ITIL 4

Una versión es una publicación de un servicio u otro elemento de configuración, o una colección de elementos de configuración, que está disponible para su uso.

Características de una Versión:

  • Incluye todos los cambios SW y HW validados
  • Preparada para instalación en entorno de producción
  • Colección coherente de elementos de configuración
  • Disponible para uso por usuarios finales
🏗️ Enfoque Tradicional

Liberaciones planificadas - Versiones grandes con ciclos de desarrollo largos.

  • Plan de liberación detallado
  • Pruebas exhaustivas pre-release
  • Documentación completa
  • Aprobación formal
⚡ Enfoque Ágil

Despliegue en pequeños incrementos - Liberaciones frecuentes y rápidas.

  • Iteraciones cortas
  • Feedback continuo
  • Mejora incremental
  • Adaptabilidad alta

🔄 Entorno DevOps

La gestión de versiones está integrada con la integración continua y las herramientas para la entrega continua.

Integración Continua

Automatización del proceso de construcción y pruebas cada vez que se realizan cambios en el código.

Entrega Continua

Capacidad de desplegar cambios de forma rápida y confiable en cualquier momento.

Despliegue Continuo

Automatización completa del pipeline desde el código hasta producción.

🎯 Beneficios de la Gestión de Versiones

  • Control de cambios: Seguimiento preciso de modificaciones
  • Trazabilidad: Historia completa de evolución del software
  • Recuperación: Capacidad de volver a versiones anteriores
  • Colaboración: Trabajo en equipo coordinado
  • Calidad: Validación y testing sistemático

📊 Definición de Despliegue

Un despliegue incluye los componentes de infraestructura y aplicaciones que posibilitan la entrega de desarrollo o mantenimiento SW, incorporando nuevas funcionalidades o correcciones.

La release posibilita la activación de las nuevas funcionalidades para los usuarios finales.

💥 Big Bang

Todo de una vez - La administración e implementación se ejecuta como un solo proceso una única vez.

⚠️ Alto riesgo, difícil rollback
🔄 Por Fases

Implementación gradual - Se fragmenta por módulos o funcionalidades implementadas y probadas.

✓ Riesgo controlado, feedback temprano
⚡ Entrega Continua

Automatización completa - En cualquier momento de manera automatizada.

🔄 Pull: Proceso detecta nueva versión
📤 Push: Servidor CI realiza despliegue

🎨 Estrategias Avanzadas de Despliegue

🔵🟢 Blue/Green Deployment

Dos entornos idénticos:

  • Blue: Se despliega nueva versión
  • Green: Se libera para usuarios
  • Cambio instantáneo entre entornos
🐤 Canary Release

Liberación gradual:

  • Se despliega para pocos usuarios
  • Incremento gradual de audiencia
  • Monitoreo continuo de métricas
  • Rollback rápido si hay problemas

🏗️ Tendencias de Despliegue

🗂️ Tradicional con Plan

Despliegue manual con documentación detallada, validaciones y aprobaciones formales.

📋 Infraestructura como Código (IaC)

Gestión automatizada de infraestructura mediante código versionado y controlado.

Ventajas de IaC:

  • Consistencia entre entornos
  • Versionado de infraestructura
  • Automatización completa
  • Recuperación rápida ante fallos

🎯 Objetivos de la Gestión de Entornos

  • Definir y normalizar los distintos entornos necesarios
  • Establecer procedimiento de despliegue y configuración
  • Mantener entornos con SW y HW necesario
  • Promocionar y desplegar versiones en cada entorno

🏗️ Pipeline de Entornos

💻 Entorno Local

Desarrollo individual del programador en su máquina personal.

🔨 Entorno Desarrollo

Integración del trabajo del equipo de desarrollo.

🔗 Entorno Integración

Pruebas de integración entre diferentes componentes.

🧪 Entorno Pruebas

Testing formal y validación de funcionalidades.

🎭 Entorno Preproducción

Staging - Réplica exacta del entorno de producción.

🚀 Entorno Producción

Sistema en vivo utilizado por usuarios finales.

⚖️ Clasificación de Entornos

🖥️ Entornos Virtuales

Ventajas:

  • Rápida creación y destrucción
  • Consistencia garantizada
  • Menor coste de infraestructura
  • Fácil clonación y backup
🏭 Entornos Reales

Características:

  • Hardware físico dedicado
  • Rendimiento nativo completo
  • Mayor fidelidad para producción
  • Coste más elevado

🔧 Mejores Prácticas

Principios Fundamentales:

  • Paridad: Entornos lo más similares posible
  • Inmutabilidad: Entornos no se modifican, se recrean
  • Versionado: Configuración de entornos versionada
  • Automatización: Despliegues y configuración automatizados
  • Monitoreo: Observabilidad en todos los entornos
  • Seguridad: Aislamiento y control de acceso
